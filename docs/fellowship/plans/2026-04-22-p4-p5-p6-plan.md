ETHOS_FILE="${PROJECT_CWD}/templates/ethos.md"

### Task A2: Session-start hook — project-local skills survey (P6)

File: `hooks/session-start` (bash)

**Add section 8** after the ethos check:

```bash
# ── 8. Project-local skills and agents survey (P6) ───────────────────────────
CLAUDE_DIR="${PROJECT_CWD}/.claude"
if [ -d "$CLAUDE_DIR" ]; then
    # Survey .claude/skills/ and .claude/agents/
    skill_count=0
    agent_count=0
    skill_names=""
    agent_names=""

    if [ -d "$CLAUDE_DIR/skills" ]; then
        while IFS= read -r name; do
            [ -n "$name" ] || continue
            skill_names="${skill_names}${name}, "
            skill_count=$((skill_count + 1))
        done < <(find "$CLAUDE_DIR/skills" -maxdepth 1 -mindepth 1 -type d -exec basename {} \; 2>/dev/null)
    fi

    if [ -d "$CLAUDE_DIR/agents" ]; then
        while IFS= read -r name; do
            [ -n "$name" ] || continue
            agent_names="${agent_names}${name}, "
            agent_count=$((agent_count + 1))
        done < <(find "$CLAUDE_DIR/agents" -maxdepth 1 -mindepth 1 -name "*.md" -exec basename {} .md \; 2>/dev/null)
    fi

    if [ $skill_count -gt 0 ] || [ $agent_count -gt 0 ]; then
        # Strip trailing ", "
        skill_names="${skill_names%, }"
        agent_names="${agent_names%, }"

        inventory="Project-local: "
        [ $skill_count -gt 0 ] && inventory="${inventory}${skill_count} skill(s) [${skill_names}]"
        [ $skill_count -gt 0 ] && [ $agent_count -gt 0 ] && inventory="${inventory}, "
        [ $agent_count -gt 0 ] && inventory="${inventory}${agent_count} agent(s) [${agent_names}]"
        inventory="${inventory}. Project-local takes precedence over plugin on name collision."

        inventory_escaped=$(escape_for_json "$inventory")
        session_context="${session_context}\\n<PROJECT_LOCAL>\\n${inventory_escaped}\\n</PROJECT_LOCAL>"
    fi
fi
```

**Verification:** Create `.claude/skills/test-skill/SKILL.md` in a test project. Session start injects `<PROJECT_LOCAL>` with `1 skill(s) [test-skill]`.

### Task A3: Session-end hook — quest-log staleness check (P5, Move 2)

File: `hooks/fellowship-session-end.mjs`

**Add a block** before the final `respond()` call. After appending the session log line, check the quest-log's `## Current` section for items marked `[x]` that haven't moved to `Recently Completed`.

```javascript
// ── Quest-log consolidation check (P5 Move 2) ──────────────────────────
const questLogPath = join(fellowshipDir, 'quest-log.md');
if (existsSync(questLogPath)) {
  try {
    const content = readFileSync(questLogPath, 'utf8');
    // Extract the ## Current section (between ## Current and the next ## heading)
    const currentMatch = content.match(/##\s+Current\s*\n([\s\S]*?)(?=\n##\s+)/);
    if (currentMatch) {
      const currentBlock = currentMatch[1];
      const completedInCurrent = (currentBlock.match(/^-\s*\[x\]/gm) || []).length;
      if (completedInCurrent > 0) {
        // Write a one-line notice to a well-known path the next session-start can surface
        const noticePath = join(fellowshipDir, '.quest-log-reminder');
        const noticeLine = `${completedInCurrent} completed item(s) in Current not yet moved to Recently Completed. Consolidate next session.\n`;
        writeFileSync(noticePath, noticeLine, 'utf8');
      }
    }
  } catch {
    // Don't crash session-end on parse failure
  }
}
```

Then extend session-start to pick up `.quest-log-reminder` if present and inject it as `<QUEST_LOG_REMINDER>`, deleting the file after reading (one-shot notice).

**Add to `hooks/session-start` section 9:**

```bash
# ── 9. Quest-log consolidation reminder (P5) ─────────────────────────────────
REMINDER="${FELLOWSHIP_DIR}/.quest-log-reminder"
if [ -f "$REMINDER" ]; then

    rm -f "$REMINDER"  # One-shot notice
fi
```

**Verification:** Mark an item `[x]` in `## Current`, end session, restart. Next session-start surfaces `<QUEST_LOG_REMINDER>`. File is deleted after read.

## Project-Local Skills and Agents

At session start, the hook surveys `.claude/skills/` and `.claude/agents/` in the project root. If present, they surface in the `<PROJECT_LOCAL>` context block.

**Precedence on name collision:** project-local wins. If a project has `.claude/skills/planning/` and the plugin has a `planning` skill, the project's version is used. The project owner knows what they need better than the plugin.

**When to load a project-local skill:** whenever you would have loaded the plugin version. The decision is the same; the source differs.

**What not to do:** do not assume the plugin set is complete. If a project has its own `.claude/agents/auth-specialist.md`, that is the intended dispatcher for auth work on that project — not Gimli with generic auth instructions.
```

Add a new section:

## Project-local skills and agents

Projects can extend Fellowship with their own skills and agents under `.claude/` in the project root:

- **`.claude/skills/{skill-name}/SKILL.md`** — project-specific skills
- **`.claude/agents/{agent-name}.md`** — project-specific agents

At session start, the Fellowship hook surveys both directories and surfaces the inventory to Gandalf. **On name collision, project-local takes precedence over plugin.** If your project has a `planning` skill, the project's version is used.

Use this when the project has patterns the plugin doesn't know about — domain terminology, specific review checklists, dispatch conventions unique to the codebase.
```
